# SDLC Document Folder

This folder keeps SDLC documents in chronological order.

## Writing Order
1. `01_project_charter.tex` (Project initiation and approval)
2. `02_project_plan.tex` (Execution, schedule, resources)
3. `03_prd.tex` (Product requirements)
4. `04_srs.tex` (Software requirements specification)
5. `05_sds.tex` (Software design specification)
6. `06_test_plan.tex` (Test strategy and quality gates)
7. `07_test_cases.tex` (Detailed test scenarios)
8. `08_rtm.tex` (Requirement traceability)
9. `09_release_deployment_plan.tex` (Release/go-live process)
10. `10_operations_maintenance_plan.tex` (Run and support model)
11. `11_post_implementation_review.tex` (Closure and lessons learned)

Build any document by opening that `.tex` file in VS Code and running LaTeX Workshop build.
